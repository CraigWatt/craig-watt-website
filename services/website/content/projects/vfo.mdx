## What vfo Does

vfo is a small C utility for batch-encoding a video library without hand-building ffmpeg commands for every file. Instead of manual queues, it looks at each file’s resolution, codec, and color space, then makes a sensible decision.

## Why I Built It

- reduce repetitive encoding work
- keep output choices predictable
- make video processing less error-prone

## How It Works

vfo inspects the file it is given, then applies rules from `vfo_config.conf` to decide which `ffmpeg` path to take.
That keeps the command-line interface simple while still letting me tune profiles for different kinds of source material.

## Practical Notes

- macOS builds are available as either a `.pkg` installer or a static ZIP
- Linux support is planned
- the behaviour is intentionally driven by configuration rather than a long list of manual flags

## Why It Matters

Manual encoding of diverse video files is time-consuming and easy to get wrong. vfo turns that into a repeatable workflow and gives me a compact example of building a useful C tool with a clean operator experience.
