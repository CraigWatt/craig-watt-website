CraigWatt.co.uk is both a portfolio and a working example of how I like to build platforms:
content first, easy to deploy, and straightforward to maintain.

It is designed to do three things well:

- present engineering work clearly
- make publishing quick without losing structure
- run as a real production system rather than a throwaway personal site

## Why I Built It

- give visitors a clear view of the work I care about
- turn project write-ups into something I can maintain myself
- keep the site small enough that it can evolve with my career

## What It Demonstrates

- **Next.js App Router and MDX** for flexible routing and content authoring
- **Content collections** for typed frontmatter and predictable publishing
- **Terraform, AWS, and GitHub Actions** for a repeatable deployment path
- **Testing and build checks** that keep the site honest as it changes
- **A contact form backed by AWS SES** so the site is a working product, not just a brochure

## Engineering Approach

The site is intentionally content-driven. I want to be able to rewrite the homepage, add a project note, or publish a new article without changing the overall shape of the app.

That means:

- the public pages stay readable and direct
- the content stays easy to update
- the infrastructure stays boring in the best possible way

## Why It Matters

This project is a good summary of how I like to work: small enough to understand, polished enough to share, and close enough to production that every choice still matters.
